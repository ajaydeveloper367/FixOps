RAG Layer — Where It Sits & What It Does

RAG (Retrieval-Augmented Generation) acts as the system's institutional memory. Instead of loading full runbooks (3,000+ tokens), the controller fetches only the 2–3 most relevant 300-token chunks and injects them into the worker's context. Result: richer investigation quality at a fraction of the token cost.

Chunk Freshness — What's Stored Per Chunk in pgvector

chunk_id
UUID. Primary key. Used to identify a chunk for deletion/update.
source_type
confluence | github_readme | rca_report | monitoring_runbook — used to filter retrieval by source category if needed.
source_id
Confluence page_id, GitHub file path, or investigation_id. Groups all chunks belonging to the same source document.
content_hash
SHA-256 of chunk text. Compared at re-index time — if unchanged, skip re-embedding. Saves ~80% of embedding API calls on weekly runs.
indexed_at
Timestamp of last embedding. Used in retrieval to optionally deprioritise very old chunks if a newer RCA exists on the same topic.
source_updated_at
Last-modified time of the source document. If source_updated_at > indexed_at, the chunk is flagged as potentially stale and gets lower retrieval weight.
entity_tags
Array of entity names this chunk is relevant to: ["dp-service", "bulkFHIR-processor", "hl7message"]. Enables targeted pre-filtering before cosine search to reduce search space.
embedding
1536-dim vector (text-embedding-3-small). Indexed with HNSW for fast approximate nearest-neighbour search at query time.
Manual re-index — available via REST endpoint and CLI for on-demand updates:
POST /rag/reindex?source=confluence&page_id=12345 — re-index a single Confluence page immediately
POST /rag/reindex?source=rca&investigation_id=inv-2026-04-20-3829 — force re-index a specific RCA
python manage.py rag reindex --all --dry-run — preview what would change without writing

Stale chunk detection — retrieval response includes a freshness_warning: true flag on any chunk where source_updated_at > indexed_at. The controller logs this and triggers a background re-index job for that source automatically.
